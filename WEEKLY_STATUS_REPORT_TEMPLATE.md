## Instructions for Completing This Report

1. **Fill in metrics first** — Get actual numbers from dashboards (GA4, Stripe, Discord API, GitHub API)
2. **Assess status** — 🟢 (on/above target), 🟡 (at risk), 🔴 (below target/blocked)
3. **Explain anomalies** — For anything red or unexpected, provide context and action
4. **Highlight learnings** — What surprised you? What changed your thinking?
5. **Be concise** — Target: 3-4 pages, not 10 pages of detail
6. **Use trends, not just snapshots** — Show direction (↑/→/↓), not just this week's number
7. **End with action** — Each problem should have a next step, not just a description
