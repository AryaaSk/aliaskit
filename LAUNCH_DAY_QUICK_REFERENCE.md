## If Something Breaks

**Website Down:**
→ Activate backup
→ Tweet: "Aware of issue, working on it"
→ Update every 15 min

**API Not Working:**
→ Product assesses scope
→ Tweet: "Investigating API issue, ETA [time]"
→ Post status page

**Press Release Wrong:**
→ Call CEO + Legal
→ Draft correction: "Clarification: [X] not [Y]"
→ Send to media contacts

**Negative Backlash:**
→ Assess: Fair or trolling?
→ If fair: "You're right, here's how we're..."
→ If trolling: Ignore, let users defend

## Key Reminders

✅ **Be transparent** — Users trust honesty
✅ **Respond fast** — Don't disappear
✅ **Acknowledge valid points** — Shows you listen
✅ **Stick to facts** — No defensiveness
✅ **Communicate progress** — "Working on it, update in 15 min"

❌ **Don't be defensive** → "You're wrong because..."
❌ **Don't go silent** → Radio silence = red flag
❌ **Don't delete criticism** → Looks like cover-up
❌ **Don't argue on Twitter** → You can't win
❌ **Don't make promises you can't keep** → Sets wrong expectation
