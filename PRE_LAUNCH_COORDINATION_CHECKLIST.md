# Pre-Launch Coordination Checklist

**Owner:** Growth Marketer
**Status:** Ready for execution (Days -3 to Day 0)
**Trigger:** Use when other teams report readiness or Day -2 arrives

---

## Overview

This checklist guides the Growth Marketer through Days -3 to Day 0, ensuring all teams are aligned, all systems are ready, and launch can proceed with confidence.

---

## Day -3 (Today or Earlier)

**DONE - All Marketing Assets Completed**
- [x] All copy finalized (HN, Twitter, LinkedIn, Product Hunt, blog, newsletters)
- [x] Design brief delivered to design team
- [x] Devrel delegation assigned (ALI-37)
- [x] Metrics dashboard setup guide provided
- [x] Contingency plans created
- [x] Weekly status template prepared

**ACTION ITEMS:**
- [ ] Post status update to ALI-34 summarizing week's deliverables
- [ ] Monitor other teams' progress (check daily)
- [ ] Identify any blockers preventing other teams from progressing

**Communication:**
- [ ] Slack message to team: "All marketing assets complete. Monitoring other teams' progress."
- [ ] Prepare to brief leadership on current status

## Day -2

**CHECKPOINT: Other Teams Making Progress?**

**Check with each team:**

### Design Team Status
- [ ] Have you started on Asset 1 & 2?
- [ ] Do you need clarifications on specs?
- [ ] What's your ETA for delivery?
- [ ] Do you have all brand assets (colors, fonts, logos)?

**If delayed:** Offer to provide brand guidelines or simplify requirements

### Product Team Status
- [ ] Demo environment progress? (% complete)
- [ ] Any blockers or issues?
- [ ] What's your ETA for validation?
- [ ] Do you need help testing?

**If delayed:** Offer contingency plan (delay launch 24h or launch without demo)

### Leadership Status
- [ ] Have you reviewed all materials?
- [ ] Any feedback on messaging?
- [ ] When can you provide final approval?
- [ ] Any messaging changes needed?

**If delayed:** Provide pre-approved positioning as fallback, escalate timeline pressure

### Analytics Team Status
- [ ] When will GA4 setup begin?
- [ ] Do you have all the implementation details?
- [ ] Any blockers or questions?
- [ ] ETA for testing?

**If delayed:** Confirm manual tracking backup is ready

### Ops/Support Status
- [ ] Support team briefed on Day 0?
- [ ] Any issues with 30-min SLA commitment?
- [ ] FAQ document ready?
- [ ] Support email monitored?

**If delayed:** Offer community support recruitment option

**ACTION ITEMS:**
- [ ] Conduct 1:1 check-ins with each team lead (by EOD)
- [ ] Create status summary of all teams
- [ ] Identify critical blockers
- [ ] Post update to ALI-34 with team status
- [ ] Escalate any critical blockers to leadership immediately

**Communication:**
- [ ] Post to Slack #launch-coordination: "Pre-launch status check: [summary of each team]"
- [ ] If any team is significantly behind: flag for leadership review

## Day -1 (Final Preparation)

**CRITICAL PHASE - Everything Must Be Ready by EOD**

### Verification Phase (Morning)

**Design Graphics:**
- [ ] Design team delivered all 5 assets (or priority 2)?
- [ ] Graphics are correct dimensions (test at 50% scale)?
- [ ] All files exported in correct formats (PNG, JPG, WebP)?
- [ ] Graphics are readable on mobile?
- [ ] Text is accurate and no typos?

**If not ready:** Activate text-based contingency (2-hour expedited version)

**Product Demo:**
- [ ] CTO confirmed demo works and is production-ready?
- [ ] Demo link tested personally (by Growth Marketer)?
- [ ] Demo completes in <30 seconds?
- [ ] All marketing CTAs pointing to correct demo link?
- [ ] Load testing completed for Day 0 traffic spike?

**If not ready:** Delay launch 24 hours or launch without demo

**Leadership Approval:**
- [ ] CMO/CTO approved all messaging?
- [ ] Any messaging changes communicated?
- [ ] Final approval given in writing (Slack/Paperclip)?
- [ ] Go/no-go decision documented?

**If not ready:** Implement contingency (use pre-approved positioning, escalate to CEO)

**Analytics GA4:**
- [ ] GA4 property created and live?
- [ ] Measurement ID added to website?
- [ ] All custom events configured and tested?
- [ ] UTM parameters tested in staging?
- [ ] Real-time dashboard created and shared?
- [ ] Slack webhook configured for alerts?

**If not ready:** Activate manual tracking backup

**Ops/Support:**
- [ ] Support team online and briefed?
- [ ] FAQ document live and linked?
- [ ] Support email monitored and responsive?
- [ ] Onboarding guide available?
- [ ] All documentation links tested?

**If not ready:** Recruit community support, adjust SLA if needed

**Devrel/Community:**
- [ ] ALI-37 (Community & Partnerships) is ready to execute?
- [ ] Framework partner outreach queued?
- [ ] Discord and GitHub infrastructure ready?
- [ ] SDK examples available?

**If ready:** Confirm readiness for Day 0/1 launch

### Alignment Phase (Afternoon)

**Final Team Standup (All Leads, 2 PM ET)**
- [ ] Schedule standup with all team leads
- [ ] Growth Marketer facilitates
- [ ] Each team confirms: ✅ Ready / ⚠️ At Risk / ❌ Blocked

**Possible Outcomes:**
- ✅ All green: Proceed with Day 0 launch
- ⚠️ 1 team at risk: Activate contingency for that item
- ❌ Multiple blockers: Escalate to leadership for launch delay decision

**Go/No-Go Decision (4 PM ET)**
- [ ] If all teams report ✅: Launch proceeds Day 0
- [ ] If any team reports ❌: Leadership makes delay decision
- [ ] Decision documented in Paperclip comment

**Communication:**
- [ ] Post to ALI-34: "Pre-launch verification complete. Status: [GO/NO-GO]"
- [ ] Slack #launch-coordination: Final status summary
- [ ] If GO: Confirm Day 0, 10 AM ET launch
- [ ] If NO-GO: Communicate new launch date to all stakeholders

## Escalation Procedures

**If critical item isn't ready by Day -1 EOD:**

1. Identify the blocker
2. Document in Slack #launch-coordination
3. Post to ALI-34 comment
4. Escalate to leadership immediately
5. Implement contingency plan
6. Communicate new timeline (if delaying) or proceed with contingency

**Escalation Chain:**
- Growth Marketer → CTO/CMO → CEO (if critical)

**Do not launch without alignment from leadership**
