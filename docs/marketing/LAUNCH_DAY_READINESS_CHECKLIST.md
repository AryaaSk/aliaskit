## Critical Issues — Escalation Path

### If Website/API Down
1. Page Engineering lead immediately
2. Activate incident response
3. Post status update on Twitter
4. Notify PH community if >15 min outage
5. Update status page

### If Analytics Not Tracking
1. Alert analytics lead
2. Check GA4 configuration
3. Verify event tags firing
4. Plan retroactive data import if needed

### If Support Volume Exceeds 30-Min SLA
1. Page support team lead
2. Alert engineering for immediate help
3. Prioritize critical bugs vs. questions
4. Consider pre-written FAQ responses

### If Negative Press/Comments
1. Alert leadership immediately
2. Don't respond defensively
3. Prepare factual counter-message
4. Prioritize customer feedback over critics

### If Signup Rate Below Target (<10/hour)
1. Alert growth team immediately
2. Review first comments/feedback
3. Check for product/UX issues
4. Prepare messaging optimization

## Red Flags (Escalation Needed 🚨)

- ⚠️ Website down or very slow
- ⚠️ API errors for new users
- ⚠️ No signups for 30+ minutes
- ⚠️ Signup/demo very confusing (common feedback)
- ⚠️ Critical product bug (blocking core functionality)
- ⚠️ Negative HN/PF comment storm
- ⚠️ Support queue growing (>10 waiting)
- ⚠️ Multiple identical complaints (systematic issue)
