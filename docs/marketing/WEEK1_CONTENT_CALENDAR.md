## Tone & Messaging Guidelines

**Messaging Pillars:**
1. **Simple:** "30 lines of code" (not "provisioning infrastructure")
2. **Practical:** Show real use cases (not abstract)
3. **Community:** Celebrate what users build
4. **Honest:** Share learnings and challenges openly

**Avoid:**
- Hype without substance
- Technical jargon (use plain language)
- Comparing to competitors directly
- Over-promising features

**Channels:ón
- **PH:** Community-first, transparent, responsive
- **HN:** Technical depth, architecture discussion
- **Twitter:** Fast, conversational, grateful
- **LinkedIn:** Professional, visionary, strategic
- **Blog:** Depth, education, SEO-friendly
