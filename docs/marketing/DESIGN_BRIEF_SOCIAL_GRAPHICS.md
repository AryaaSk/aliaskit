## Overview

This brief specifies 5 core social media graphics for AliasKit's launch. Each graphic has specific use cases, dimensions, and messaging requirements.

**Design Philosophy:**
- Clean, technical aesthetic (not playful or casual)
- Agent-focused imagery (not human-centric)
- Dark/light mode variants (for different contexts)

## Asset 1: "One API Call" Hero Graphic

**Purpose:** Hero graphic for Twitter, LinkedIn, Product Hunt. Creates visual impact in feed.

**Use Cases:**
- Twitter thread header (pinned tweet)
- LinkedIn post cover image
- Product Hunt thumbnail (if applicable)
- Blog post featured image (backup)

**Dimensions & Formats:**
- Primary: 1200 x 628 px (Twitter OG, LinkedIn)
- Secondary: 1080 x 1080 px (Instagram, mobile)
- Secondary: 1080 x 1920 px (Instagram Stories)
- Formats: PNG (quality) + JPEG (size) + WebP (modern)

**Content Direction:**

**Left Side (Visual Element):**
- Code snippet: 5-10 lines of clean, readable code
- Language: JavaScript or Python (whichever looks cleaner)
- Example code:
  ```javascript
  const identity = await aliaskit.provision({
    agentId: "my-agent",
    channels: ["email", "sms"]
  });
  ```
- Style: Monospace font, subtle syntax highlighting
- Background: Dark code editor aesthetic

**Right Side (Agent Element):**
- Simple agent avatar/icon (robot or abstract circular shape)
- Below avatar: Email address + phone number icons
- Visual treatment: Clean, minimal, professional
- Color: Accent color (CTA highlight)

**Center/Overlay:**
- Headline: "One API Call"
- Subheading: "Complete agent identity"
- Visual: Arrow from code to agent (suggesting provisioning)

**Color Palette:**
- Background: Dark (navy/charcoal) or white depending on variant
- Code: Standard terminal colors (green/blue syntax)
- Accent: Brand primary color (for CTA, highlights)
- Text: White on dark, black on light

**Do's:**
- ✅ Make code readable (test at 50% size)
- ✅ Use existing AliasKit brand colors
- ✅ Keep design minimal (not busy)
- ✅ Ensure text is readable at thumbnail size

**Don'ts:**
- ❌ Don't make it too "AI/sci-fi" looking
- ❌ Don't use generic stock photos
- ❌ Don't make code hard to read
- ❌ Don't introduce new brand colors

## Asset 2: Comparison Table Graphic

**Purpose:** Show AliasKit advantage vs. alternatives (Twilio+SendGrid, AgentMail).

**Use Cases:**
- Twitter thread (Day 2 follow-up)
- LinkedIn carousel post
- Blog post comparison section
- Newsletter sponsorship creative (optional)

**Dimensions & Formats:**
- Primary: 1080 x 1080 px (Instagram, LinkedIn carousel)
- Secondary: 1200 x 628 px (Twitter)
- Formats: PNG + JPEG + WebP

**Content Direction:**

**Table Format:**
- 4 columns: Dimension | AliasKit | Twilio+SendGrid | AgentMail
- 5 rows: Setup Time | Cost/Month | API Complexity | Phone Support | Single API

**Data to Display:**
| Dimension | AliasKit | Twilio+SendGrid | AgentMail |
|-----------|----------|-----------------|-----------|
| Setup Time | 5 min | Weeks | Days |
| Cost | $50/mo | $150+ | $40+/mo |
| API Complexity | Simple (1 call) | Complex (3+ APIs) | Medium |
| Complete Solution | Email + Phone | Scattered | Email only |
| Single Dashboard | ✅ | ❌ | ✅ |

**Visual Treatment:**
- Header row: Brand primary color background, white text
- AliasKit column: Highlight (light green or accent color)
- Alternating row colors for readability
- Large, readable font (test at 50% scale)
- Simple checkmarks (✅) vs. X (❌) for comparison

**Tone:**
- Professional, not aggressive
- Present facts, not claims
- Emphasis on ease and bundling, not price alone

**Do's:**
- ✅ Use real, defensible data
- ✅ Make AliasKit clearly the winner
- ✅ Use consistent icon styling
- ✅ Ensure text is 100% readable at thumbnail size

**Don'ts:**
- ❌ Don't be dishonest or exaggerate
- ❌ Don't make competitors look bad (just look good in comparison)
- ❌ Don't use tiny fonts
- ❌ Don't crowd the table

## Asset 3: Use Case Carousel (3 Cards)

**Purpose:** Show real-world use cases for agent identity. One card per use case, designed as carousel.

**Use Cases:**
- LinkedIn carousel post (Days 2-3)
- Twitter thread (if 3-part thread)
- Discord/blog (as image sequence)

**Dimensions & Formats:**
- 1080 x 1080 px each (Instagram, LinkedIn carousel, Discord)
- Formats: PNG + JPEG + WebP
- All 3 cards should be visually consistent

**Card Structure (Each Card):**

**Top Section (Use Case Title):**
- Large, bold headline (one per card)
- Example: "Hiring Agent", "Support Agent", "Workflow Agent"
- Background: Subtle gradient (use brand colors)

**Middle Section (Visual Element):**
- Icon or simple illustration representing the use case
- Hiring agent: Briefcase + checkmark icon
- Support agent: Headset + message icon
- Workflow agent: Gears/automation icon
- Style: Minimalist, professional
- Color: Brand accent color

**Bottom Section (Problem/Solution):**
- Problem statement: "Can't verify identities" / "Can't sign up for services" / etc.
- Arrow or divider
- Solution: "AliasKit solves this" + brief benefit (one sentence)

**Design Notes:**
- Each card should look good standalone AND as a series
- Consistent layout and typography across all 3
- Readable at mobile size
- Professional but approachable

**Card Content:**

**Card 1: Hiring Agent**
- Icon: Briefcase or building
- Problem: "Can't verify legitimacy with job boards"
- Solution: "Real identity enables independent sourcing"

**Card 2: Support Agent**
- Icon: Headset or chat
- Problem: "Can't verify customer identity via email/SMS"
- Solution: "Verified agent identity builds trust"

**Card 3: Workflow Agent**
- Icon: Gears or automation
- Problem: "Can't sign up for services autonomously"
- Solution: "Complete identity enables service integration"

**Do's:**
- ✅ Keep all 3 cards visually consistent
- ✅ Use icons consistently across series
- ✅ Make each use case relatable
- ✅ Ensure text is readable at mobile size

**Don'ts:**
- ❌ Don't make cards look like totally different designs
- ❌ Don't use stock photos
- ❌ Don't make text too small
- ❌ Don't use more than 2 colors (brand colors)

- User quote/testimonial (1-2 sentences max)
- Large, readable font
- Professional tone

- Company/role (optional)

**Design Notes:**

- Maintain white space and readability
- Professional, not cheesy
- Subtle texture/gradient OK, but not overdone

**Do's:**
- ✅ Make quote prominent and readable
- ✅ Use brand colors subtly
- ✅ Leave room for avatar photo
- ✅ Ensure clean, professional look

**Don'ts:**
- ❌ Don't make it look like a generic testimonial template
- ❌ Don't use decorative fonts
- ❌ Don't overcrowd with visual elements
- ❌ Don't make the quote too long
